## What this is

ShockRehearsal is a deterministic, stdlib-only tool that lets operators run a "tabletop rehearsal" before a high-stakes allocation decision (priority flow rights, demand bloc volume, refusal capacity).

It ingests a shock scenario (rights profile + bloc volume + refusal curves + coercion vector) and produces a clear 3-way verdict plus an auditable, hash-chained ledger entry.

## Verdict scheme

```text
passed → gaps → failed
```

- **passed** — rehearsal demonstrates sufficient rights + volume + refusal headroom under the simulated shock
- **gaps** — partial coverage (one axis thin but not critical)
- **failed** — one or more axes clearly insufficient

## CLI triplet

```bash
python shock_rehearsal.py sample
python shock_rehearsal.py run examples/shock_passed.json --format md
python shock_rehearsal.py report examples/shock_gaps.json
```

## Quick start

```bash
cd ShockRehearsal
python shock_rehearsal.py sample > my_shock.json
python shock_rehearsal.py run my_shock.json --format markdown
python shock_rehearsal.py run examples/shock_failed.json --format json
```

## Dual output

Every `run` produces both machine JSON and human Markdown:
- JSON: verdict, per-axis scores, ledger reference
- Markdown: formatted report with boundary notice and ledger hash

## Boundary

> This is **not** a procurement auction or financial option exchange. It is a rehearsal instrument only.

## Design origin

Generated by the `recreate` methodology (run `004-shock-rehearsal`) from the `recreate_prj` corpus.

Reused (with explicit costs):
- ReserveFlow (priority clearing + ledger)
- BuyBloc (demand bloc volume)
- RefusalOption (refusal headroom pricing)

Lenses applied: L1_DirectionReversal, L17_ConstraintSubstitute.
